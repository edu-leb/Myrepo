Проект представляет собой backend‑часть интернет‑магазина книг, реализованную на NestJS + TypeORM + PostgreSQL.

В проекте выполнены требования из ТЗ:

CRUD‑модули Auth, Users, Authors, Books, Orders
Каждый модуль содержит: entity, controller, service, DTO, валидацию, обработку ошибок
Реляции One‑to‑Many(Users - Orders, Authors - Books) и Many‑to‑Many(Orders - Books)
JWT‑аутентификация, роли user/admin
Профиль текущего пользователя
Ограничение доступа по ролям
Исправление маршрутов и логики, связанных с авторизацией(регистрация пользователя, логин, хранение пароля в виде хеша, JWT‑токен)

Админ‑маршруты:
GET /users
GET /users/:id
PATCH /users/:id
DELETE /users/:id
POST /users

Исправлена логика создания заказа (текущий пользователь берётся из токена)

Ограничен доступ к заказам
Пользователь: видит только свои заказы, может отменить только свой заказ, не может открыть, изменить или удалить чужой заказ.
Администратор: видит все заказы, может управлять любым заказом.

Ограничены маршруты Authors и Books
Только admin может: создавать автора, обновлять автора, удалять автора, создавать книгу, обновлять книгу, удалять книгу.

Реализована смена пароля
Маршрут: PATCH /auth/change-password
Логика: проверка старого пароля, хеширование нового, запрет на повтор старого пароля, доступ только по токену.
